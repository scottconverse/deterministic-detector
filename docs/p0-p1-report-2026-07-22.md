## Defects found and fixed during review (before first push of code)

1. **Silent vacuous pass in `ci/detectors.yml`:** appending a generated
   `[tool.mutmut]` block to a pyproject.toml that already has one produces
   invalid TOML; `|| true` would have swallowed the parse failure into a green
   job with an empty survivor list. Now detected and loudly SKIPPED in the job
   summary. Run log tail is also surfaced whenever `mutmut results` fails.
2. **Hook BOM fragility:** `json.load(sys.stdin)` raised on a UTF-8 BOM and
   the intentional swallow made it silent. Now decodes `utf-8-sig`. Found
   because the red test refused to go red — the make-it-fail-first rule doing
   its job.
3. **Hook crash path:** `subprocess.run` `OSError` now degrades to exit 0
   (the hook must never break editing).

## Facts that changed the plan

- mutmut's Windows failure is **structural** (POSIX-only stdlib import), not a
  soft guard. The optional local native lane is dead; WSL is the only local
  option and is strictly optional (plan D4 stands, stronger than assumed).
- Local plugin install needs no marketplace and no CLI: a folder (or junction)
  under `~\.claude\skills\` containing `.claude-plugin\plugin.json` auto-loads
  at session start.
